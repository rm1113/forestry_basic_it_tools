# forest-science-tools-course

Учебный репозиторий для вводного семинара по повседневным IT-инструментам для студентов лесного профиля.

## Как использовать этот репозиторий

1. Склонируйте репозиторий и создайте свою ветку для работы
2. Следуйте инструкциям в `README.md` для выполнения всех заданий
3. Вы можете проверять свою работу локально через команду `pytest`
4. Сделайте `push` в свою ветку, чтобы запустить автоматическую проверку через GitHub Actions
5. Откройте Pull Request, чтобы преподаватель мог проверить вашу работу и дать обратную связь

## Структура репозитория

- `README.md` — инструкция для студентов
- `requirements.txt` — Python зависимости курса
- `data/trees.csv` — входные данные
- `notebooks/01_forest_analysis.ipynb` — ноутбук с заданиями
- `results/` — сюда нужно сохранить итоговый `csv`
- `tests/` — автоматические проверки

## Задание

### 1. Установить Python

Рекомендуемая версия: **Python 3.14**. Для установки можно использовать официальный сайт [python.org](https://www.python.org/downloads/) или Microsoft Store (для Windows).

Откройте терминал и проверьте, что вы находитесь в корне проекта (где лежит `README.md`):

На Windows:

```powershell
cd
```

Linux / macOS:

```bash
pwd
```

Вы должны увидеть что-то вроде `\path\to\GitHub\forestry_basic_it_tools`.

Убедитесь, что Python установлен и доступен в терминале. Введите и убедитесь, что команда возвращает версию Python:

```bash
python --version
```


### 2. [опционально] Создать виртуальное окружение и установить зависимости

Стандартный способ для изоляции зависимостей в Python — это виртуальное окружение. Оно позволяет не мешать глобальным пакетам и легко управлять зависимостями проекта. Стандартный инструмент для этого — `venv`, который входит в стандартную библиотеку Python. Но мы рекомендуем использовать [`uv`](https://docs.astral.sh/uv/). 

Установка библиотек происходит с помощью `pip`, инструмента для управления пакетами в Python.

```bash
pip install uv
```

После этого можно создать виртуальное окружение:

```bash
forestry_basic_it_tools> uv venv
# Using CPython 3.13.1
# Creating virtual environment at: .venv
# Activate with: .venv\Scripts\activate
```

### 3. Установка зависимостей

Для этого проекта нам понадобится 3 библиотеки:
- `pandas` для работы с данными;
- `pytest` для автоматической проверки;
- `jupyter` для работы с ноутбуками.

Зависимости описаны в файле `requirements.txt`. Чтобы их установить, выполните команду:

```bash
uv pip install -r requirements.txt
# или 
pip install -r requirements.txt
```

Теперь можно сделать первый запуск теста, чтобы убедиться, что все пакеты установлены:

```bash
pytest tests/test_0.py
```

### 3. Запустить Jupyter Notebook

Из корня проекта:

```bash
jupyter notebook
```

Откройте файл и выполните задания в ноутбуке:

```text
notebooks/01_forest_analysis.ipynb
```

### 4. Выполнить задание в ноутбуке

После выполнения задания вторая часть тестов будет проходить успешно:

```bash
pytest tests/test_1.py
```

### 5. Git-задание

Теперь вам нужно сделать коммит с решением. Используйте GitHub Desktop или командную строку. Добавьте к нему короткий комментарий,вроде "Solutuon for forest analysis". После этого запушьте изменения в свою ветку и откройте Pull Request, чтобы преподаватель мог проверить вашу работу. 

Попробуйте запустить все тесты, командой 

```bash
pytest tests/test_2.py
```

Первый тест должен пройти успешно, а второй — нет.

#### Учимся делать revert

Сделайте любое ненуженое изменение, например тут:

> Меняю

Сделайте коммит с этим сохраненными изменением и запушьте его. Убедитесь, что второй тест все еще не проходит. Теперь нужно сделать revert этого коммита, чтобы вернуть всё в исходное состояние. В GitHub Desktop это делается через меню "History" → "Revert commit". После этого запушьте изменения и убедитесь, что все тесты проходят успешно.

Если все сделано правильно, то тесты должны проходить локально после команды

```bash
pytest
```

А в гитхабе напротив последнего коммита загорит зеленая галочка. Значит все сделано правильно! Не забудьте открыть Pull Request, чтобы получиить обратную связь.
